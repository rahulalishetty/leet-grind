# 535. Encode and Decode TinyURL

## Problem Description

This problem is a companion problem to the system design question **Design TinyURL**.

TinyURL is a URL shortening service. The idea is that instead of sharing a long URL, the system generates a short URL that redirects to the original one.

For example:

```
Original URL:
https://leetcode.com/problems/design-tinyurl

Tiny URL:
http://tinyurl.com/4e9iAk
```

Your task is to design a class that can:

- encode a long URL into a shortened URL
- decode the shortened URL back to the original URL

There is **no restriction on how the encoding/decoding algorithm works**.

The only requirement is:

> A URL encoded by the system must be correctly decoded back to its original URL.

---

# Class Design

Implement the `Solution` class with the following methods.

## Constructor

```
Solution()
```

Initializes the object of the system.

---

## encode

```
String encode(String longUrl)
```

Takes a **long URL** and returns a **shortened tiny URL**.

---

## decode

```
String decode(String shortUrl)
```

Takes a **tiny URL** and returns the **original long URL**.

It is guaranteed that the `shortUrl` was generated by the same object.

---

# Example

## Input

```
url = "https://leetcode.com/problems/design-tinyurl"
```

## Output

```
"https://leetcode.com/problems/design-tinyurl"
```

## Explanation

```
Solution obj = new Solution();

string tiny = obj.encode(url);
// returns the encoded tiny URL

string ans = obj.decode(tiny);
// returns the original URL after decoding it
```

---

# Constraints

```
1 <= url.length <= 10^4
url is guaranteed to be a valid URL
```
