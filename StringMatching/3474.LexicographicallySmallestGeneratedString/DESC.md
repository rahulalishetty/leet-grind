# 3474. Lexicographically Smallest Generated String

## Problem Statement

You are given two strings:

- `str1` of length `n`
- `str2` of length `m`

A string `word` of length **`n + m - 1`** is defined to be generated by `str1` and `str2` if it satisfies the following conditions for each index `0 ≤ i ≤ n - 1`:

- If `str1[i] == 'T'`, then the substring of `word` of length `m` starting at index `i` **must equal** `str2`

  ```
  word[i .. i + m - 1] == str2
  ```

- If `str1[i] == 'F'`, then the substring of `word` of length `m` starting at index `i` **must NOT equal** `str2`

  ```
  word[i .. i + m - 1] != str2
  ```

Return the **lexicographically smallest possible string** that can be generated by `str1` and `str2`.

If no such string exists, return:

```
""
```

---

# Example 1

### Input

```
str1 = "TFTF"
str2 = "ab"
```

### Output

```
"ababa"
```

### Explanation

The string `"ababa"` satisfies the conditions:

| Index | T/F | Substring of length m |
| ----- | --- | --------------------- |
| 0     | T   | "ab"                  |
| 1     | F   | "ba"                  |
| 2     | T   | "ab"                  |
| 3     | F   | "ba"                  |

Both `"ababa"` and `"ababb"` satisfy the constraints.

However:

```
"ababa" < "ababb"
```

So we return:

```
"ababa"
```

---

# Example 2

### Input

```
str1 = "TFTF"
str2 = "abc"
```

### Output

```
""
```

### Explanation

No string exists that satisfies all required substring constraints.

---

# Example 3

### Input

```
str1 = "F"
str2 = "d"
```

### Output

```
"a"
```

### Explanation

The only restriction is that the substring must **not equal `"d"`**.

The lexicographically smallest character satisfying this is:

```
"a"
```

---

# Constraints

- `1 ≤ n == str1.length ≤ 10^4`
- `1 ≤ m == str2.length ≤ 500`
- `str1` consists only of `'T'` or `'F'`
- `str2` consists only of lowercase English letters
